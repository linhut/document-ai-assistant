## 问题1：模板下载按钮 ✅

### 需求
为每个预设模板添加下载按钮，下载已设置样式格式的模板文档。

### 实现方案

#### 后端 API
**文件**：`backend/api/routes/template_download.py`

**功能**：
- 读取模板规则配置
- 生成带样式的 Word 文档
- 应用字体、字号、对齐等格式
- 返回文件供下载

**API 端点**：
```
GET /api/template/download/{template_id}
```

**生成内容**：
- 标题（应用配置样式）
- 正文示例（3段，应用配置样式）
- 页面设置（边距、纸张大小）

#### 前端按钮
**文件**：`frontend/src/pages/Templates.tsx`

**位置**：每个模板卡片中间

**功能**：
- 点击触发下载
- 调用后端 API
- 自动下载 .docx 文件
- 文件名：`{模板名}模板.docx`

### 用户操作
1. 进入模板中心
2. 找到需要的模板（如"通知"）
3. 点击"下载模板"按钮
4. 自动下载格式化的 Word 文档
5. 打开文档开始编辑

### 模板内容示例
```
==================
      通知       （方正小标宋简体 22pt 居中）
==================

这是正文第一段的示例内容。请在此处填写您的实际内容。
                              （仿宋_GB2312 16pt 首行缩进2字符）

这是正文第二段的示例内容。本模板已按照规范要求配置了
字体、字号、行距等格式。

您可以直接在此模板基础上编辑内容，格式将自动符合公文
规范要求。
```

### 当前状态

- ✅ 后端正常运行：`http://127.0.0.1:8765`
- ✅ API 端点全部可用

### 排查建议

#### 1. 检查浏览器控制台
按 F12 打开开发者工具，查看：
- Console 标签：是否有 JavaScript 错误
- Network 标签：API 请求是否成功
- Elements 标签：HTML 是否正常渲染

#### 2. 常见原因
- **路由问题**：刷新页面后路由丢失
- **API 请求失败**：网络错误或后端未响应
- **组件渲染错误**：React 组件报错
- **缓存问题**：浏览器缓存旧代码

#### 3. 解决方法
```bash
# 方法1：强制刷新
Ctrl + Shift + R（清除缓存并刷新）

# 方法2：清除浏览器缓存
设置 → 隐私 → 清除浏览数据

# 方法3：重启前端
cd frontend
npm run dev
```
