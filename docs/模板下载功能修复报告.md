## 🐛 问题描述

**错误信息**：
```
1 validation error for ParagraphFormat
space_after_pt
  Input should be a valid number, unable to parse string as a number
  [type=float_parsing, input_value='20pt', input_type=str]
```

---

## 🔍 问题分析

### 根本原因
YAML规则文件中使用了 `spacing_after: 20pt` 字符串格式，但Pydantic模型期望的是数字类型。

### 影响范围
- 15个规则文件
- 8种公文模板下载

### 错误日志
```
ERROR official_doc_ai - Download template notice failed: 
1 validation error for ParagraphFormat
space_after_pt
  Input should be a valid number
```

---

## ✅ 修复方案

### 修改内容
将所有规则文件中的：
```yaml
spacing_after: 20pt  # 错误：字符串
```

改为：
```yaml
spacing_after: 20    # 正确：数字
```

### 1. 类型一致性
**问题**：YAML中混用了字符串和数字类型  
**解决**：统一使用数字类型，在代码中解析字符串

### 2. 数据验证
**问题**：Pydantic模型严格验证类型

## 🔧 技术细节

### 修复脚本
```python
import re
with open('file.yaml', 'r', encoding='utf-8') as f:
    content = f.read()
content = re.sub(r'spacing_after:\s*(\d+)pt', r'spacing_after: \1', content)
with open('file.yaml', 'w', encoding='utf-8') as f:
    f.write(content)
```

### 规则格式
```yaml
# 正确的格式
title:
  font: "方正小标宋简体"
  size: 22pt              # 字符串 - 需要解析
  align: center
  spacing_after: 20       # 数字 - 直接使用
```
