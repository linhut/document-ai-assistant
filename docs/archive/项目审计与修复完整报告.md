## 🐛 问题1：文档处理失败 - ✅ 已修复

### 问题描述
用户在文档处理模块上传文档后点击"开始检查"提示"处理失败，请重试"。

### 根因分析
**文件**：`frontend/src/api/client.ts`（第31行）

API 响应拦截器已经返回了 `response.data`：
```typescript
apiClient.interceptors.response.use(
  (response) => {
    return response.data;  // ← 这里已经返回了 data
  },
  ...
);
```

但所有页面组件仍然在访问 `response.data.xxx`，导致访问 `undefined.xxx` 失败。

### 受影响文件
1. `frontend/src/pages/DocumentProcess.tsx` - 第82行
2. `frontend/src/pages/CheckCenter.tsx` - 第59、78行
3. `frontend/src/pages/AISettings.tsx` - 第41-44、69-73、102行
4. `frontend/src/pages/Templates.tsx` - 第36、48行
5. `frontend/src/pages/Rules.tsx` - 第33、44行

### 修复内容
将所有 `response.data.xxx` 改为 `response.xxx`：

**修复示例**：
```typescript
// ❌ 错误（修复前）
const uploadedDocId = uploadResponse.data.id;

// ✅ 正确（修复后）
const uploadedDocId = uploadResponse.id;
```

✅ 结果显示功能正常
