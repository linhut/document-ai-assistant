## 🎯 功能概述

**问题**：模板中心的模板无法下载  
**需求**：利用规则配置生成标准的公文模板文档供用户下载  
**解决方案**：基于YAML规则动态生成符合规范的.docx模板文件

---

## ✅ 实现功能

### 1. 模板生成引擎
**位置**：`backend/api/routes/template_download.py`

**核心功能**：
- 读取YAML规则配置
- 根据规则生成标准格式的文档
- 为每种公文类型提供示例内容
- 应用正确的字体、字号、对齐方式

### 2. 支持的模板类型

| 序号 | 模板类型 | ID | 段落数 | 状态 |
|------|---------|-----|--------|------|
| 1 | 通知 | notice | 16 | ✅ |
| 2 | 请示 | request | 16 | ✅ |
| 3 | 报告 | report | 18 | ✅ |
| 4 | 会议纪要 | meeting | 20 | ✅ |
| 5 | 函 | letter | 11 | ✅ |
| 6 | 决定 | decision | 11 | ✅ |
| 7 | 通告 | announcement | 11 | ✅ |
| 8 | 公告 | notice_public | 11 | ✅ |

**所有模板文件大小**：约35KB

### 3. 模板内容示例

#### 通知模板
```
标题：关于XXX的通知
主送机关：各部门、各单位:
正文：
  为进一步做好XXX工作,现将有关事项通知如下:
  一、工作目标
  （具体内容）
  ...
落款：（单位名称）
日期：XXXX年XX月XX日
```

#### 会议纪要模板
```
标题：XXX会议纪要
时间: XXXX年XX月XX日XX时
地点: XXX会议室
主持人: XXX
参会人员: XXX、XXX、XXX等XX人
会议主要内容:
  一、XXX议题
  （具体内容）
  ...
会议决定:
  一、（决定事项）
  ...
```

## 🔧 技术实现

### 1. 动态内容生成
```python
def _get_template_content(template_id: str, template_name: str) -> dict:
    """
    根据不同公文类型返回相应的示例内容
    """
    templates = {
        "notice": {...},
        "request": {...},
        "meeting": {...},
        ...
    }
    return templates.get(template_id, default)
```

### 2. 格式应用
```python
# 标题格式
- 字体：方正小标宋简体
- 字号：22pt
- 对齐：居中

# 正文格式
- 字体：仿宋_GB2312
- 字号：16pt
- 首行缩进：2字符
- 行距：28.95pt
- 对齐：两端对齐

# 落款格式
- 对齐：右对齐
```

### 3. 页面设置
```python
PageSetup(
    paper_width_mm=210,    # A4纸宽度
    paper_height_mm=297,   # A4纸高度
    margin_top_mm=37,      # 上边距
    margin_bottom_mm=35,   # 下边距
    margin_left_mm=28,     # 左边距
    margin_right_mm=26     # 右边距
)
```
